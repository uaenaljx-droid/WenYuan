# Product

## Register

product

## Users

This project is for readers, writers, students, and humanities-curious visitors who want to browse literary, philosophical, and intellectual figures through a quiet spatial archive. They arrive in an exploratory mood rather than a task-heavy workflow, often looking for orientation, comparison, or a memorable entry point into a person's works and temperament.

## Product Purpose

The product presents a curated 3D persona atlas on a realistic Earth. Success means the user first feels they are entering a cinematic documentary globe, then can click into a clear encyclopedia-style profile for a specific person. The product should foreground people, ideas, routes, and reliable attribution while hiding engineering details from the main interface.

## Brand Personality

Restrained, archival, cinematic. The voice is calm and editorial, with short labels and plain language. The emotional goal is intellectual trust: the page should feel like a museum interface for literature and thought, not a technical demo.

## Anti-references

Do not make this feel like an AI dashboard, GitHub data viewer, agent directory, social-media creator wall, neon sci-fi control panel, or generic glassmorphism landing page. Avoid purple triangle markers, loud gradient cards, source-path displays, README/path labels, framework labels, debug counts as decoration, and any self-media, EQ, ancient-text, or multi-agent system as a primary display object.

## Design Principles

1. People before plumbing: literary and intellectual figures are the only main content, with data provenance available only through a low-key source dialog.
2. Documentary before dashboard: the default state should be sparse, realistic, and camera-led, with UI that supports the globe rather than competing with it.
3. Archive before spectacle: the clicked state should feel like a reliable profile poster, using verified fields, compact routes, and readable prose.
4. Trust over coverage: show fewer figures with stronger references rather than padding the atlas with uncertain or irrelevant entries.
5. Motion as narration: route playback and focus movement should feel like a guided tour, not a fast carousel or attention grab.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Support keyboard search, Escape to close dialogs, arrow-key navigation, reduced-motion preferences, mobile touch interaction, and readable Chinese copy. When portraits are not verifiably licensed or sourced, use sober text placeholders instead of AI-generated avatars.
